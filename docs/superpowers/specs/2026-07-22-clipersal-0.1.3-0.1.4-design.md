# Clipersal 0.1.3-beta + 0.1.4-beta — design spec

Date: 2026-07-22. Author: autonomous session (user away; all open questions
self-answered per explicit instruction). Status: approved by default (auto mode);
implementation follows the wave plan at the bottom.

Context: Clipersal 0.1.2-beta is a stable, tested (557 tests) PySide6 instant-replay
tool. These two releases make it feel like a daily-driver app: first a UI/UX +
quality-of-life release (0.1.3), then a creator-features release (0.1.4). No new
third-party runtime dependencies in 0.1.3; 0.1.4 re-includes the already-shipped
PySide6 QtMultimedia submodule (currently excluded only in the PyInstaller spec) —
that is a packaging change, not a new dependency.

## Hard conventions every change must respect

- New `Config` fields default to pre-existing behavior (old config.json →
  byte-identical ffmpeg argv). Gate every new ffmpeg arg behind `value != default`.
- Persisted settings: full wiring checklist — `Config` field → `PERSISTED_KEYS` +
  `_KEY_TYPES` (+ `_NONEABLE_KEYS` if nullable) → argparse flag seeded from
  persisted value → `config_from_args` → `apply_settings` (read from `new_values`,
  diff, live-mutate vs capture-restart via `dataclasses.replace` + `resolve_setup`
  before mutating, assignment, `save_overrides` dict) → Settings UI sends the key.
- Every ffmpeg/ffprobe subprocess spreads `**NO_WINDOW_KWARGS`.
- Never touch Qt from a non-GUI thread — `AppSignals` / worker `Signal`s only.
- Theme tokens read at call time (`theme.ACCENT`), never by-value import.
- Every trigger goes through the IPC boundary (`ipc_client.send_command`).
- Per-clip metadata and caches live in sibling files, never config.json.
- Qt tests: `pytest.importorskip("PySide6")` + `QT_QPA_PLATFORM=offscreen` preamble,
  monkeypatch dialogs/ffmpeg, pump `sendPostedEvents()`, never inject real keys.

# 0.1.3-beta — "Experience update"

## 1. Session stats via IPC (`STATS`)

New IPC command `STATS` (registered in `cli.py`, added to `trigger.py` choices as
`stats`) returns a single-line `key=value` pipe-separated payload:

`state=RECORDING|uptime=123.4|segments=27|buffer_bytes=12345678|encoder=h264_nvenc|buffer_seconds=60|clips_free_bytes=123456789|clips_count=5`

- `state`: existing STATUS string (RECORDING/PAUSED/CRASHED).
- `uptime`: seconds since current capture `start()` — new
  `SegmentedCapture.uptime_seconds() -> float | None` (None when not running);
  store `time.monotonic()` in `start()`, clear in `stop()`.
- `segments` / `buffer_bytes`: count + summed size of `seg-*.ts` in buffer dir
  (vanish-tolerant stat).
- `encoder`: `state.setup.encoder`.
- `clips_free_bytes`: `shutil.disk_usage(clips_dir).free` — feeds the low-disk
  banner with no extra thread (the window already polls status every 1.5 s).
- `clips_count`: number of `*.mp4` in clips dir.
- Unknown/missing values degrade to empty strings; parser must tolerate that.

Existing `STATUS` stays unchanged (tray depends on it).

## 2. Home dashboard

Status card gains: pause/resume toggle button whose label follows polled state
("Pause capture" / "Resume capture"), and a stats line fed by `STATS` (uptime as
H:MM:SS, buffer fill `segments × segment_seconds` vs `buffer_seconds`, buffer MB,
encoder, free disk GB). Quick actions stay: Save now (primary), Save last 15 s /
30 s / 60 s buttons (IPC `SAVE <n>`).

New dismissible **crash banner** (same pattern as `_update_banner`): shown when
state == CRASHED, with a "Restart capture" button → IPC `RESUME` (which recovers
CRASHED by design) and a "View logs" button. Auto-hides when state recovers.

New **low-disk banner**: shown when `clips_free_bytes` < 1 GiB (module constant),
auto-hides when above 1.5 GiB (hysteresis, no flapping). Not configurable (YAGNI).

Window title carries state: `"Clipersal"` / `"Clipersal — Paused"` /
`"Clipersal — Capture stopped"`.

## 3. Gallery overhaul (`gallery_window_qt.py`)

- Header gains: search `QLineEdit` (substring filter on filename, case-insensitive),
  sort `QComboBox` (Newest first / Oldest first / Name A–Z / Largest first),
  and a "Select" toggle button entering selection mode. Filter/sort state stored on
  the frame and re-applied in `refresh()`; enumeration stays `clips_newest_first()`
  (filtering happens after — that function is shared with Home's recent strip).
- Sorting by size/age uses stat (already fetched per row); name sort is free.
  No duration sort (probe cost).
- Selection mode: rows show a checkbox; header shows "Select all" / "Clear" and a
  "Delete selected (N)" danger action with one `QMessageBox.question` confirm;
  per-file `unlink()` failures log + continue (existing `_do_delete` convention).
  Selection state is by path and vanish-tolerant.
- Row meta line gains clip duration (`MM:SS`) probed async by the existing
  `ThumbnailWorker` shape (extend it or add a sibling `DurationWorker` —
  ffprobe `get_duration_seconds`, None → omitted, never crash).
- Row right-click context menu (`QMenu`): Open / Play externally, Reveal in folder,
  Favorite (star toggle), Rename…, Trim…, Copy path (clipboard), Delete.
  Keep the existing 5 per-row buttons — the menu is additive.
- **Favorites**: star button per row (★/☆), persisted via new
  `clip_metadata.py` sidecar (`clips_dir/.clipmeta.json`, schema below); favorite
  rows get a small accent star; a "Favorites first" checkbox in the header sorts
  them to the top within the current sort.
- Footer status line: "N clips · X.Y GB total".
- Retention protection: `concat.enforce_clip_retention` gains optional
  `protected: set[str] | None = None` (clip filenames, not stems, to stay exact);
  cli's save/apply paths pass current favorites. Default None = old behavior.

### `clip_metadata.py` (new module)

Sidecar JSON at `clips_dir/.clipmeta.json`: `{"version": 1, "clips": {"<filename>":
{"favorite": true, "note": "..."}}}`. API:
`load_metadata(clips_dir) -> dict`, `is_favorite(clips_dir, name) -> bool`,
`set_favorite(clips_dir, name, fav: bool)`, `favorites(clips_dir) -> set[str]`,
`set_note(clips_dir, name, note)`, `note_for(clips_dir, name) -> str`,
`prune(clips_dir, existing_names)` (drop entries for deleted clips; called on
gallery refresh). Atomic tmp+replace writes; corrupt file → treated as empty;
never raises. (Notes are consumed by 0.1.4's details dialog; the file ships in
0.1.3 with favorites.)

## 4. Quick-save hotkeys + screenshot hotkey

- `HotkeyListener` extended to accept `dict[str, Callable]` of combo→callback
  (single-combo constructor preserved for back-compat; it wraps into the dict).
- New persisted Config fields: `quick_save_hotkey_1` ("" = disabled),
  `quick_save_seconds_1` (30), `quick_save_hotkey_2` (""), `quick_save_seconds_2`
  (60), `screenshot_hotkey` (""). Empty string never reaches pynput
  (validation: `is_valid_combo` on non-empty; duplicates of each other or the main
  combo rejected in `apply_settings` with an error string).
- `rebind_hotkey()` builds one `GlobalHotKeys` map: main combo → SAVE, quick-save
  combos → `SAVE <seconds>`, screenshot combo → `SCREENSHOT` (all via
  `ipc_client.send_command`, matching the existing round-trip rule).
- Settings → "Save & hotkey" card gains two quick-save rows (HotkeyField +
  seconds spinbox 5–300) and one screenshot HotkeyField.

## 5. Screenshot capture

New module `screenshots.py`:
`save_screenshot(ffmpeg_path, buffer_dir, clips_dir) -> Path` —
picks the newest *finalized* segment (sorted `seg-*.ts`, drop the still-growing
last one — deliberately not a second live capture device: on Windows ddagrab/gdigrab
conflicts and on Wayland it's impossible without a portal round-trip; the frame is
at most ~2×segment_seconds old, which matches the instant-replay mental model),
grabs its last frame (`-sseof -0.3`, fallback `-ss 0`) to
`clips_dir/screenshot-{date}-{time}.png` (unique-suffixed; PNG so the gallery's
`*.mp4` glob and the retention sweep ignore it). Raises `EmptyBufferError` when
nothing is finalized; partial output unlinked on failure.

IPC `SCREENSHOT` (save_lock-serialized, returns path); trigger command
`screenshot`; tray item "Take screenshot"; Home quick-action button; optional
hotkey (§4). Toast reuse: `show_save_toast` gains an optional `title` parameter
("Screenshot saved").

## 6. Framerate persistence + resolution scale

- `framerate` becomes persisted (currently CLI-only): `PERSISTED_KEYS` +
  `_KEY_TYPES` entry, argparse default seeds from persisted, `apply_settings`
  treats it as capture-restart-affecting. Settings → Capture card: FPS combo
  (15 / 24 / 30 / 60). Default 30 = old argv.
- New `resolution_scale`: `"native"` (default) | `"1080p"` | `"720p"`. When not
  native, `capture._build_command` appends `scale=-2:<height>` to the software
  filter chain (after the encoder's existing format fragment; for VAAPI the scale
  must happen before `hwupload`). Same restart path as bitrate. Settings combo
  next to FPS. Wayland: applied in the rawvideo filter chain the same way.

## 7. Toast upgrade (`toast_qt.py`)

- Action buttons row: **Open** (plays the file with the OS default app — new
  `open_file(path)` helper in `tray.py` next to `open_folder`, using
  `os.startfile` / `xdg-open` / `open`) and **Show in folder**
  (`reveal_in_file_manager`). Whole-toast click still opens the folder only when
  no button was clicked (buttons stop propagation).
- **Stacking**: module-level list of live toasts; each new toast offsets upward by
  (height + 12 px) per already-open toast; re-flow on close. Never-raise contract
  of `show_save_toast` preserved.
- Meta line: duration + size when cheaply known (passed in by the caller; cli
  already knows the path — probe async like the thumbnail fetch; None → omit).

## 8. Logs tab upgrade + diagnostics export

- Search box (substring, case-insensitive), level filter combo (All / INFO /
  WARNING / ERROR), auto-scroll `ToggleSwitch` (default on), "Copy" button
  (clipboard), all operating on the same 2 s tail poll.
- "Export diagnostics…" button → new `diagnostics.py`:
  `export_diagnostics_zip(target_path, log_dir, config_path, buffer_dir,
  extra: dict[str, str]) -> Path` — stdlib `zipfile` bundle of `clipersal.log*`,
  `ffmpeg.log` if present, redacted config (drop paths? no — keep paths, they're
  the point of diagnostics; note it in README), `system.txt` (OS, session type,
  Python, `ffmpeg -version` first line, encoder, monitor list). Never raises —
  returns None on total failure, partial zip best-effort. Saved via
  `QFileDialog.getSaveFileName` defaulting to `~/Desktop/clipersal-diagnostics.zip`.

## 9. Main-window keyboard shortcuts

`QShortcut`s on `MainWindow` (window-local, `Qt.WindowShortcut`): `Ctrl+S` save
now, `Ctrl+Shift+S` save last 30 s, `Ctrl+P` pause/resume toggle, `F5` refresh
gallery, `Ctrl+,` jump to Settings, `Ctrl+1..4` switch tabs. All SAVE-class
actions reuse the existing worker-thread IPC path.

## 10. Settings additions

- "Check for updates" row gains a "Check now" button + last-checked label (reads
  the update cache; triggers the same `update_check` flow on a thread, bypassing
  the 24 h throttle via a new `force=True` parameter on `check_for_update_once`).
- FPS combo + resolution-scale combo (§6), quick-save hotkey rows (§4).
- All new fields flow through the standard `apply_settings` machinery.

## 11. Tray

Menu gains "Take screenshot" (IPC `SCREENSHOT`, same worker/balloon pattern as
Save) between "Save last 30s" and "View clips".

# 0.1.4-beta — "Creator update"

## 1. In-app player (`player_qt.py`, new)

`PlayerDialog(QDialog)`: `QMediaPlayer` + `QAudioOutput` + `QVideoWidget` from
`PySide6.QtMultimedia` (+ `QtMultimediaWidgets`). Controls: play/pause, seek
slider with `current / total` time labels, volume slider, speed segmented
(0.5×/1×/1.5×/2×), and a trim row: "Set start" / "Set end" from the playhead +
"Export trim" → existing `concat.trim_clip` (stream copy) → success message +
gallery refresh. Escape closes; deleting the file while open is tolerated.
Import guarded: if QtMultimedia is unavailable, callers fall back to
`open_file()` (OS default player). Entry points: double-click a gallery row,
context-menu "Play", toast "Open" stays external (explicit user choice), player
"Play in-app" is the gallery default for double-click.

Packaging: `packaging/clipersal.spec` removes `PySide6.QtMultimedia` (and
`QtMultimediaWidgets`) from the exclude list. Size cost accepted; note in
ARCHITECTURE.md.

## 2. Export studio (`export.py`, new)

- `export_gif(ffmpeg_path, clip_path, out_dir, start=0.0, duration=3.0, fps=12,
  width=480) -> Path` — two-pass palettegen/paletteuse, palette in a temp file,
  `<stem>.gif` unique-suffixed, partial cleanup, 120 s timeout.
- `compress_clip(ffmpeg_path, encoder, clip_path, out_dir, bitrate="4M",
  scale_height=None) -> Path` — re-encode via `encoder_global_args` +
  `encoder_output_args`, `-c:a copy`, optional `scale=-2:<h>`,
  `<stem>-compressed.mp4` unique-suffixed.
- Gallery context menu gains "Export as GIF…" and "Compress…" — small modal
  dialogs (start/duration/fps/width; bitrate/scale) following the `TrimDialog`
  worker pattern; results land next to the clip and refresh the gallery (GIFs
  don't show — they're not `.mp4`; toast/message tells the path).

## 3. Clip details dialog

Context-menu "Details…": modal showing thumbnail, filename, size, mtime, duration,
resolution (new `thumbnails.get_video_info(ffprobe_path, clip_path) ->
(duration, width, height) | None`, worker-probed), favorite toggle, and an
editable note (`QPlainTextEdit`, saved to `.clipmeta.json` on dialog accept).
Notes surface as a row tooltip in the gallery.

## 4. Recording overlay (`overlay_qt.py`, new)

Frameless `Qt.Tool | Qt.WindowStaysOnTopHint | Qt.FramelessWindowHint` +
`WA_TransparentForMouseEvents` + translucent background widget: status dot +
"REC"/"PAUSED" text on a small rounded chip. Config: `overlay_enabled`
(False default), `overlay_corner` ("top-right" default; four corners, 16 px
margin). MainWindow updates it from its existing status poll (same GUI thread —
no new signals needed). Live show/hide/reposition from `apply_settings`.
Settings → Appearance card gains the toggle + corner combo.

## 5. Save sound

Config `save_sound_enabled` (False default) → on save success (GUI toast path)
`QApplication.beep()`. Settings toggle in "Save & hotkey" card. Cross-platform,
zero deps.

## 6. Clips folder size cap

Config `clips_max_gb` (0 = unlimited, default). `concat.enforce_size_cap(
clips_dir, max_bytes, protected: set[str] | None = None) -> list[Path]` — deletes
oldest non-protected `*.mp4` until total ≤ cap; favorites protected; never deletes
the clip just saved in the same call (pass it as protected). Runs in `handle_save`
after retention and on `apply_settings`. Settings slider 0–50 GB in "Clip
management".

## 7. Gallery extras

- Drag-and-drop: dragging a row out of the gallery produces file URLs
  (`QMimeData`) so clips can be dropped into Explorer/Finder/chat apps.
- Double-click plays in-app (§1). Note from details dialog shows as tooltip.

# Wave plan

Sequential waves (shared files force order); each wave = one coder subagent with a
scoped prompt, ending green on its targeted tests; I run the full suite between
waves and after each release's content freeze.

0.1.3: W1 backend core (clip_metadata, retention `protected`, capture uptime) →
W2 backend capture/config (STATS, screenshots, framerate persist, resolution scale,
quick-save hotkeys, trigger) → W3 gallery overhaul → W4 shell UI (Home dashboard,
toast, logs+diagnostics, settings fields, tray, shortcuts) → release tasks
(versions 0.1.3-beta, CHANGELOG, README, AGENTS.md).

0.1.4: W5 backend (export.py, get_video_info, size cap, new config fields +
apply wiring) → W6 player + gallery creator integration → W7 overlay + save sound
+ spec change → release tasks (0.1.4-beta, docs).
